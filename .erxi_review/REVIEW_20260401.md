## 🔴 高威胁问题（需立即修复）

### 1. 全局状态并发问题

**文件**：`backend/api/routes.py:17`

```python
cp_engine = CPEngine()  # 全局可变状态！
last_update_time = None
```

**问题**：在高并发请求下，`cp_engine.stocks.clear()` 可能导致数据竞争。多个请求同时刷新时可能互相覆盖。

**建议**：
- 使用 `threading.Lock` 保护临界区
- 或使用异步队列处理数据刷新
- 或使用单例模式配合实例级锁

---

### 2. CORS配置过于宽松

**文件**：`backend/api/main.py:23-29`

```python
allow_origins=["*"],  # 生产环境禁止！
```

**问题**：允许所有来源的跨域请求，存在安全风险

**建议**：限制为特定的前端域名
```python
allow_origins=["http://localhost:5173", "http://your-domain.com"],
```

---

### 3. 批量处理无超时保护

**文件**：`backend/data/fetcher.py:536-543`

```python
for i in range(0, len(sample), batch_size):
    ...
    time.sleep(0.05)  # 无超时控制
```

**问题**：如果数据源响应慢，可能导致请求堆积

**建议**：
- 添加异步处理和超时控制
- 使用 `asyncio` 替代同步循环
- 添加请求超时 `timeout=10`

### 6. stock_dicts 索引同步问题

**文件**：`backend/api/routes.py:75-76`

```python
for i, s in enumerate(cp_engine.stocks):
    stock_dicts[i]["total_cp"] = s.total_cp
```

**问题**：`stock_dicts` 的顺序可能与 `cp_engine.stocks` 不一致

**建议**：使用字典映射代替索引
```python
stock_map = {s.code: s for s in cp_engine.stocks}
for stock_dict in stock_dicts:
    if stock_dict['code'] in stock_map:
        stock_dict["total_cp"] = stock_map[stock_dict['code']].total_cp
```

---

### 8. 错误处理可更友好

部分异常仅打印到 console，未返回友好错误信息

**建议**：统一错误响应格式
```python
return {"error": "数据刷新失败，请稍后重试", "code": "REFRESH_FAILED"}
```

---

### 9. 缓存未考虑内存压力

大量股票数据缓存在内存中，无清理机制

**建议**：
- 添加缓存大小限制
- 使用 LRU 缓存策略
- 定期清理过期数据
