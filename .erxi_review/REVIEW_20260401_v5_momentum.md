**项目名称**：TradeSnake - 股市贪吃蛇
**文档时间**：2026-04-01
**文档类型**：战力公式优化建议

---

## 📋 背景

当前战力公式 v14 采用基本面为核心的架构：

```
总战力 = (成长分×30% + 价值分×25% + 质量分×20% + 动量分×15%) × 风险调整因子
```

其中**动量分**目前只使用**当日涨跌幅**，信息量有限。

---

## 🎯 优化目标

在保持"基本面为核心"的前提下，**小幅增强动量分的信息捕捉能力**。

---

## 📊 爱尔希的建议

### 建议方案：扩展动量分

**当前**：
```
动量分(15%) = 当日涨跌幅（限制在-10到10之间）
```

**优化后**：
```
动量分(15%) = 近5日累计涨幅×5% + 成交量变化×5% + 当日涨跌幅×5%
```

---

### 具体实现建议

```python
# 在 cp_engine.py 中修改

def calculate_momentum(self, change_pct: float, recent_change_5d: float = 0, volume_ratio: float = 1.0) -> float:
    """
    计算动量分

    参数:
    - change_pct: 当日涨跌幅 (%)
    - recent_change_5d: 近5日累计涨跌幅 (%)，默认0
    - volume_ratio: 成交量比值（当日成交量/近5日平均成交量），默认1.0
    """

    # 1. 当日动量（限制在-10到10）
    daily_score = max(-10, min(10, change_pct))

    # 2. 近5日累计动量（限制在-20到20）
    recent_score = max(-20, min(20, recent_change_5d)) * 0.5  # 折半处理

    # 3. 成交量变化（>1表示放量，<1表示缩量）
    # 放量上涨=积极信号，放量下跌=消极信号
    if volume_ratio > 1.2:
        volume_score = 5  # 放量
    elif volume_ratio < 0.8:
        volume_score = -3  # 缩量
    else:
        volume_score = 0  # 正常

    # 综合动量分（当日5% + 近5日5% + 成交量5%）
    momentum_score = daily_score * 0.05 + recent_score * 0.05 + volume_score * 0.05

    return momentum_score
```

---

## ⚠️ 不建议加入的内容

| 类型 | 原因 |
|------|------|
| MACD、KDJ等技术指标 | 指标太多会让战力值复杂且不稳定 |
| 布林带、压力位支撑位 | 需要历史数据存储成本高 |
| K线形态识别（如头肩顶） | 算法复杂，不适合简单量化 |
| 短线交易信号 | 违背"价值投资"初心 |

## 📈 预期效果

| 效果 | 说明 |
|------|------|
| ✅ 捕捉短期趋势 | 近5日涨跌比只看当日更全面 |
| ✅ 反映市场情绪 | 放量vs缩量是重要信号 |
| ✅ 战力值更稳定 | 多日平均比单日波动小 |
| ✅ 保持简单性 | 实现复杂度低，易于理解和调整 |

3. **权重调整**：如果新公式效果更好，可以适当提高动量分权重（但建议不超过25%）
4. **测试验证**：对比优化前后的战力榜单变化

## 💬 补充说明

**为什么要保持简单？**

1. **战力公式的初心**：是帮助理解股票价值，不是预测价格
2. **信息过载的风险**：太多指标会让战力值失去焦点
3. **稳定性**：简单公式的战力值更稳定，更适合长期跟踪

**什么时候可以加入更多K线？**

- 当主公验证了当前公式的有效性后
- 当有更多历史数据积累后
- 当发现某些K线信号确实能提升战力值准确性时
