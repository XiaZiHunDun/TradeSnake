## 📋 问题背景

用户指出当前战力系统存在重大逻辑缺陷：

1. **现金未纳入战力体系** - 金钱应该视为"特殊股票"
2. **换股路径缺失** - 当前是直接换，实际应该"卖→钱→买"
3. **手续费缺失** - A股交易有明确费率规则
4. **最低消费影响** - 小资金换股成本比例更高

## 📊 A股交易费用明细（2024年最新）

### 费用构成

| 费用类型 | 费率 | 收取方式 | 收取方 |
|----------|------|----------|--------|
| **印花税** | 0.05% (万五) | 仅卖出时单边收取 | 国家税务局 |
| **佣金** | 0.03% (万三，默认) | 买卖双向收取 | 券商 |
| **过户费** | 0.001% (十万分之一) | 买卖双向收取 | 中国结算 |
| **规费** | ~0.004% | 买卖双向收取 | 已含在佣金中 |

### 完整换股一次成本（买→卖）

```
卖出A：
  佣金：成交金额 × 0.03%
  过户费：成交金额 × 0.001%
  印花税：成交金额 × 0.05%
  小计：0.081%

买入B：
  佣金：成交金额 × 0.03%
  过户费：成交金额 × 0.001%
  小计：0.031%

总计：≈ 0.112%（约112元/10万元）
```

### 最低消费影响

| 交易金额 | 按0.112%计算 | 实际最低 | 多付比例 |
|----------|--------------|----------|----------|
| 1万元 | 11.2元 | 5+5=10元 | -10.7% |
| 2万元 | 22.4元 | 5+5=10元 | -55.4% |
| 5万元 | 56元 | 5+5=10元 | -82.1% |
| 10万元 | 112元 | 5+5=10元 | -91.1% |

**结论**：资金越小，实际费率越高！5元最低消费导致小资金换股极不划算。

## 🎯 升级方案设计

### 方案一：简化版（推荐先行）

**核心思路**：将现金视为"战力为0"的基准

```python
# 新增换股成本常量
TRADE_COST = {
    'commission': 0.0003,      # 佣金 0.03%
    'stamp_tax': 0.0005,       # 印花税 0.05% (仅卖)
    'transfer_fee': 0.00001,   # 过户费 0.001%
    'min_commission': 5.0,     # 最低佣金 5元/笔
}

# 完整换股一次的成本比率
TOTAL_SELL_COST = 0.0003 + 0.0005 + 0.00001  # 0.081%
TOTAL_BUY_COST = 0.0003 + 0.00001             # 0.031%
TOTAL_TRADE_COST = TOTAL_SELL_COST + TOTAL_BUY_COST  # 0.112%

# 最小有意义交易量（使最低消费影响可忽略）
MIN_TRADE_VALUE = 50000  # 5万以上的交易，最低消费影响 < 0.02%
```

**现金基准战力定义**：
- 现金无风险，但也无收益
- 设定现金的"机会成本战力" = 无风险年化收益率 / 365 × 持有天数
- 假设无风险收益率 2%/年，持有1天 = 0.0055%/天的机会成本

### 方案二：完整版（后续演进）

```python
class CashCP:
    """现金战力计算"""
    risk_free_rate = 0.02  # 年化无风险利率 2%

    @staticmethod
    def get_daily_opportunity_cost(cash: float, days: int) -> float:
        """持有现金的每日机会成本"""
        return cash * (CashCP.risk_free_rate / 365) * days

class TradeDecision:
    """换股决策引擎"""

    @staticmethod
    def should_swap(stock_a: StockCP, stock_b: StockCP,
                    value: float = 100000) -> dict:
        """
        判断是否应该从股票A换到股票B

        返回：
        - swap_profit: 换股预期收益
        - trade_cost: 交易成本
        - net_profit: 净收益
        - should_swap: 是否应该换
        """
        # 1. 计算战力差值
        cp_diff = stock_b.total_cp - stock_a.total_cp

        # 2. 战力差转换为预期收益
        # 假设CP差1分，对应年化收益差异1%
        expected_return = cp_diff * 0.01

        # 3. 计算交易成本
        sell_value = value
        buy_value = value * (1 - TOTAL_TRADE_COST)
        trade_cost = value * TOTAL_TRADE_COST + 10  # 预估滑点

        # 4. 净收益
        gross_profit = sell_value * expected_return
        net_profit = gross_profit - trade_cost

        return {
            'cp_diff': cp_diff,
            'expected_return': expected_return,
            'trade_cost': trade_cost,
            'net_profit': net_profit,
            'should_swap': net_profit > 0
        }
```

## 📈 战力公式 v15 建议

### 核心改动

1. **新增现金基准战力**
   ```
   现金CP = 无风险日收益 × 持有天数
   ```

2. **新增换股决策因子**
   ```
   换股净收益 = (B战力 - A战力) × 本金 - 交易成本
   ```

3. **战力门槛**
   ```
   只有当 战力差 × 本金 > 交易成本 时，换股才有意义
   ```

### 战力差阈值计算

| 资金量 | 交易成本 | 需要战力差(假设1分=1%) |
|--------|----------|------------------------|
| 1万 | ~10元 | > 10% |
| 5万 | ~56元 | > 1.1% |
| 10万 | ~112元 | > 1.1% |
| 50万 | ~560元 | > 1.1% |

**结论**：资金越大，换股成本占比越低，战力差阈值越小
