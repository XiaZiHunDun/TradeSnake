# 战力引擎

> **【重要更新 2026-05-06】本文档描述的 v19.8 权重已废弃，当前使用 v21 权重体系。**
>
> **v21 权重**：growth=50%, momentum=28%, value=0%, quality=5%, real_time=2%, risk_penalty=10%（乘数）
>
> **依据**：基于 414 天 Alpha 分析（2024-04-15~2026-04-23），growth IC=+0.0104(t=2.46) 唯一显著正因子，value/quality 均为反转或无效因子。
>
> 实际权重配置见 `backend/engine/cp_engine/constants.py` 中的 `WEIGHTS` 字典。

> 本文档是 ENGINE_ARCHITECTURE.md 的补充，详细描述战力引擎的设计。

---

## 一、概述

### 1.1 设计原则

- **战力为核心**：计算股票综合战力评分
- **保持原样**：v19.8 战力引擎保持原有结构，几乎不做改动
- **每日一次**：每日收盘后执行一次计算

### 1.2 引擎职责

战力引擎是分析引擎的核心模块，负责计算股票综合战力评分，为 recommender 提供选股依据。

**核心功能**：
- 多维度战力评分计算（成长、价值、质量、动量）
- 风险评估与仓位建议（Kelly公式）
- 换股决策分析（交易成本建模）
- 数据校验与清洗（拦截ST/*ST股）
- 技术指标集成（MA、MACD、RSI等）

**更新频率**：
- **日常计算**：每日收盘后执行一次完整战力计算
- **实时因子**：仅核心池股票在盘中实时更新（每分钟）
- **风险评估**：与战力计算同步更新

### 1.3 触发频率

| 触发方式 | 执行时间 | 说明 |
|----------|----------|------|
| **每日定时** | 收盘后（约15:30-16:00） | 每日仅触发一次，使用当日数据计算战力 |
| **事件触发** | 股票晋级到核心池时 | 优先计算新晋级股票战力 |
| **手动触发** | 用户请求时 | 立即计算指定股票战力 |

### 1.4 持久化策略

| 项目 | 说明 |
|------|------|
| **存储内容** | 每日战力结果（code, total_cp, 各维度分数, rank, timestamp） |
| **存储位置** | `data_manager/cp_history_store.py`（SQLite WAL模式） |
| **存储周期** | 最近2年，历史数据用于回测和分析 |
| **数据量估算** | 2000股票 × 730天 × ~500B ≈ 7MB/年 |

### 1.5 回测支持

| 项目 | 说明 |
|------|------|
| **回测目标** | 验证战力选股策略有效性 |
| **回测逻辑** | 每日战力计算后记录，对比N日后是否跑赢市场 |
| **评估指标** | 高战力组超额收益、夏普比率、最大回撤 |
| **回测触发** | 手动触发或每周定时 |

## 五、战力计算逻辑

### 5.1 总分公式（v21 当前使用）

```
总战力 = (成长分×50% + 价值分×0% + 质量分×5% + 动量分×28% + 实时分×2%) × 风险调整因子
```

> **注意**：v21 权重总和为 85%（growth 50% + momentum 28% + value 0% + quality 5% + real_time 2%），风险惩罚因子为独立乘数（0.90~1.00）。

### 5.2 各维度计算（v21）

| 维度 | 权重 | 计算思路 |
|------|------|----------|
| **成长分** | 50% | 净利润增速(60%) + 营收增速(40%)，限制在合理范围后加权 |
| **价值分** | 0% | ROE基础分 + PE健康度评分 + PEG估值评分 + PB市净率评分×0.3（v21 清零，IC=-0.009 反转因子） |
| **质量分** | 5% | 现金流评分 + 毛利率评分 + 资产负债率评分（v21 调降至最低，IC=-0.003 无效） |
| **动量分** | 28% | 多日动量(60%) + 当日涨跌幅(40%)，归一化时加权混合；原始分为纯粹的当日涨跌幅(change_pct)（v21 调增，IC=+0.009 边界显著） |
| **实时分** | 2% | 仅核心池股票，基于分钟K线的MA5/MA15变化率（注：当前版本保留未启用，作为未来功能预留） |
| **风险惩罚** | 10%（乘数） | `(1 - risk_score/100) × 0.10` 作为乘数作用于总分，非加性权重 |

**风险因子明细**（用于计算 risk_score）：

| 风险因子 | 权重 | 评分规则 |
|----------|------|----------|
| PE风险 | 35% | <0:100, >100:80, >50:50, <5:30, else:10 |
| ROE风险 | 25% | <0:100, <5:50, <10:20, else:5 |
| 涨跌幅风险 | 10% | \|change_pct\|>8:100, >5:60, >3:30, else:10 |  注：使用当日涨跌幅，非长期波动率 |
| 成长风险 | 20% | <-50:100, <-20:70, <0:40, >100:30, else:10 |
| 营收风险 | 10% | <-30:80, <-10:50, <0:20, else:5 |

**最终风险分**：`risk_score = min(100, 0.4 × max(各因子) + 0.6 × 加权平均)`

### 5.3 v19.8 权重（已废弃，仅作历史参考）

> 以下为 v19.8 版本的权重配置，已于 2026-05-06 废弃。

```
总战力 = (成长分×30% + 价值分×25% + 质量分×20% + 动量分×8% + 实时分×2%) × 风险调整因子
```

| 维度 | 权重（v19.8） | v21 权重 | 变化说明 |
|------|---------------|----------|----------|
| 成长分 | 30% | 50% | +20%，唯一显著正 alpha 因子 |
| 价值分 | 25% | 0% | -25%，IC 反转为负 |
| 质量分 | 20% | 5% | -15%，IC 无效 |
| 动量分 | 8% | 28% | +20%，边界显著 |
| 实时分 | 2% | 2% | 不变 |
| **总计** | **85%** | **85%** | 权重总和不变 |


### 5.4 稳健归一化

使用**97.5%百分位裁剪**避免极值干扰：
- clip_percentile=0.975 时，裁剪范围 [1.25%, 98.75%]，即裁剪掉最高和最低各 1.25% 的数据（共 2.5%）
- 再进行标准化到 0-100 分
- 小数据集(<20)使用 IQR 方法避免百分位不准确

### 5.5 行业相对PE调整

- 计算各行业PE中位数
- 根据个股PE与行业中位数的比值调整风险和价值分：

**风险调整**：

| PE比值（个股/行业中位数） | 风险加成 |
|--------------------------|----------|
| > 2.5倍 | +25 |
| > 2.0倍 | +20 |
| > 1.5倍 | +10 |
| < 0.4倍 | -15 |
| < 0.5倍 | -10 |
| < 0.7倍 | -5 |
| 其他 | 0 |

**价值加成**：当PE比值 < 0.6倍时，value_score 获得额外加成

### 5.6 涨跌停判断（仅沪深主板适用）

> **产品范围说明**：本系统战力计算与交易分析仅覆盖**沪深主板**（不含创业板/科创板/北交所）。下表为沪深主板的涨跌停阈值。

| 板块 | 涨跌停阈值 | 说明 |
|------|-----------|------|
| 主板（沪市/深市） | 10% | 标准涨跌幅 |

> **实现**：`cp_engine.py` 中 `is_limit_up()` 函数根据股票代码（60/00/30/688前缀）判断板块，30/688前缀股票不在本产品分析范围内。

## 六、关键模块逻辑

### 6.1 DataValidator（数据校验）

**校验流程**：
1. **ST/*ST股拦截**：名称以ST/*ST等开头的股票直接拒绝
2. **必需字段检查**：code, name, price, pe, roe等必须存在
3. **数值范围限制**：各字段限制在合理范围内，超出则截断

### 6.2 TradeDecision（换股决策）

**核心公式**：
```
换股净收益 = (B战力 - A战力) × 本金 × 持有天数 - 交易成本
```

**决策阈值**：
- 净收益 > 交易成本×120% → 强烈建议换股
- 净收益 > 0 → 可以考虑换股
- 净收益 > -50%交易成本 → 持有不动
- 净收益 < -50%交易成本 → 别换！

**交易成本构成**：
- 券商佣金：万分之三（双向）
- 印花税：万分之五（仅卖出）
- 过户费：十万分之一（仅沪市双向）

### 6.3 KellyCalculator（仓位计算）

**Kelly公式**：f* = p - q/b
- f* = 最佳下注比例
- p = 获胜概率
- q = 失败概率 (1-p)
- b = 盈亏比

**A股改进**：
- 使用半Kelly（50%保守系数）
- 最大仓位限制20%
- 考虑交易成本影响

### 6.4 RiskAnalyzer（风险评估）

| 风险类型 | 评估内容 |
|----------|----------|
| 仓位集中度 | 单只股票占比是否过高（>50%为高风险） |
| 行业集中度 | 单一行业占比是否过高（>60%为高风险） |
| 流动性风险 | 大持仓股票是否存在变现困难 |
| 市场系统性风险 | 市场整体CP偏低时（<40）谨慎操作 |
| 财报季风险 | 4、7、10月为财报发布期，波动较大 |
| 小账户风险 | 资金<5000时，佣金影响过大 |

> **实现说明**：实际实现中 (`risk_analyzer.py`) 阈值略有调整：
> - 仓位集中度：`high>70%`, `medium>50%`, `low>30%`（设计为>50%高风险）
> - 行业集中度：`high>60%`, `medium>40%`（与设计一致）

### 6.5 实时因子（real_time_score）

仅核心池股票在盘中计算，基于分钟级数据：
- MA5变化率 × 50%
- MA15变化率 × 30%
- 成交量异常度 × 20%

## 七、刷新策略

### 7.1 刷新间隔

| 时间段 | 刷新间隔 |
|--------|----------|
| 交易时间（9:30-11:30, 13:00-15:00） | 60秒 |
| 盘前盘后（8:00-20:00） | 300秒 |
| 其他时间 | 3600秒 |

### 7.2 市场阶段

| 阶段 | 时间 | 说明 |
|------|------|------|
| pre_open | < 9:30 | 开盘前 |
| morning_open | 9:30-10:00 | 早盘开盘 |
| morning | 10:00-11:00 | 早盘 |
| late_morning | 11:00-11:30 | 早盘尾段 |
| noon_break | 11:30-13:00 | 午间休市 |
| afternoon | 13:00-15:00 | 下午盘 |
| market_close | 15:00-16:00 | 收盘 |
| after_hours | 16:00-20:00 | 盘后 |
| closed | 其他 | 非交易时段 |

> **实现说明**：实际实现中 (`trading_time.py`) 将市场阶段简化为4种状态：
> - `pre_open`: 9:30之前
> - `trading`: 交易时段（早盘+下午盘合并）
> - `lunch_break`: 午间休市
> - `closed`: 收盘后及其他时段
>
> 刷新间隔主要由**小时**决定，因此9阶段 vs 4状态的差异不影响核心功能。
