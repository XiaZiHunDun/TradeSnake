## 二、数据源评估

### 2.1 候选数据源对比

| 维度 | AKShare | Tushare | 东方财富/同花顺爬取 |
|------|---------|---------|------------------|
| **费用** | 完全免费 | 基础免费，高级接口需积分（积分可通过社区贡献获取） | 免费但需自行维护爬虫 |
| **安装** | `pip install akshare` | `pip install tushare`，需注册获取 token | 无标准库，需自写 |
| **财务报表** | ✅ 有（利润表/资产负债表/现金流量表） | ✅ 有，且更结构化 | ✅ 有但需爬取 |
| **日线行情** | ✅ 有 | ✅ 有 | ✅ 有 |
| **PE/PB** | ✅ 有（个股+行业） | ✅ 有 | ✅ 有 |
| **行业分类** | ✅ 申万行业分类 | ✅ 申万行业分类 | 需爬取 |
| **沪深300成分股** | ✅ 当前成分股 | ✅ 当前+历史变动 | 需爬取 |
| **历史成分股快照** | ⚠️ 无直接接口 | ✅ 有 `index_weight` 接口 | 需爬取 |
| **财报发布日期** | ⚠️ 无直接接口 | ✅ 有 `disclosure_date` 字段 | 不确定 |
| **资金流向** | ✅ 有（个股+板块） | ✅ 有 | ✅ 有 |
| **无风险利率** | ✅ 有（国债收益率曲线） | ✅ 有 | 需爬取 |
| **接口稳定性** | ⚠️ 依赖上游网站，偶有变动 | ✅ 自有服务器，较稳定 | ❌ 爬虫随时可能失效 |
| **数据质量** | 良好，但个别字段需校验 | 良好，数据治理更规范 | 取决于爬取质量 |
| **限流** | 无明确限流，但建议控制频率 | 有积分对应的调用额度 | 有反爬风险 |

### 2.2 关键差异分析

**AKShare 的优势**：完全免费、零门槛、覆盖面广，适合 MVP 快速验证。

**AKShare 的短板**（对我们项目有影响的）：

| 短板 | 影响 | 严重程度 |
|------|------|---------|
| 无历史成分股快照 | V4 回测的幸存者偏差无法消除 | **致命**（回测结果不可信） |
| 无财报发布日期 | V4 回测的前视偏差无法消除 | **致命**（回测结果不可信） |
| 接口依赖上游网站 | 上游改版后接口可能暂时失效 | 中等（可通过缓存缓解） |

**Tushare 的优势**：数据治理好、有历史成分股和财报发布日期——这两个是我们回测的硬性需求。

**Tushare 的短板**：需要积分。基础接口免费（200 积分起步），高级接口（如历史成分股权重）需要 2000+ 积分。积分获取方式：注册送积分、完善信息送积分、社区贡献送积分。

### 2.3 AI 增强层数据源评估

| 数据 | 来源 | 费用 | 接口方式 | 备注 |
|------|------|------|---------|------|
| 财报 PDF 全文 | 巨潮资讯网 | 免费 | HTTP 下载 | 标准公开渠道，无需爬虫 |
| 个股公告全文 | 巨潮资讯网公告 API | 免费 | REST API | 按股票代码查询 |
| 财经新闻流 | 财联社快讯 | 免费 | RSS / 网页 | 高频更新，需本地 LLM 过滤 |
| 财经新闻流 | 新浪财经 | 免费 | RSS | 覆盖面广 |
| LLM 推理（财报解读） | 云端 API（GPT/Claude/通义千问） | 按调用计费 | REST API | 低频（一季度一次 × 300 只），年成本约 ¥200-500 |
| LLM 推理（事件感知） | 本地开源模型（Qwen/DeepSeek） | 免费（需 GPU） | 本地调用 | 高频任务，避免 API 成本 |

> **成本评估**：财报解读按每份财报约 50 页文本、GPT-4 级别 API 计算，每只股票每季度约 ¥0.3-0.5，300 只 × 4 季度 ≈ ¥400-600/年。如使用国产 API（通义千问/DeepSeek API），成本可降至 1/3。

## 四、关键问题与风险

### 4.1 致命问题：AKShare 无法独立支撑 V4 回测

| 缺失数据 | 影响 | 结论 |
|---------|------|------|
| 历史成分股快照 | 回测无法消除幸存者偏差 | **必须引入 Tushare** |
| 财报发布日期 | 回测无法消除前视偏差 | **必须引入 Tushare** |

如果只用 AKShare，V1-V3（数据验证、战力计算、人工校验）可以完成，但 **V4 回测无法做到设计文档要求的标准**。

### 4.2 数据质量风险

| 风险 | 说明 | 应对 |
|------|------|------|
| 财务数据缺失 | 个别股票某些季度的财务指标可能为空 | 缺失字段用行业均值填充，标注"数据修复" |
| AKShare 字段格式不一致 | 部分接口返回字符串（如 "12.34%"），需清洗为数值 | 数据层统一清洗 |
| AKShare 接口变动 | 依赖上游网站（东方财富/新浪等），上游改版后可能报错 | DataProvider 抽象接口，随时切换 |
| 历史 PE/PB 数据深度 | 计算历史分位需要至少 3 年 PE 数据，部分次新股不足 | 不足 3 年的用已有数据计算，标注"历史数据不充分" |
| 自由现金流口径 | "资本支出"在不同数据源中字段名不同，口径可能有差异 | V1 阶段专项验证，抽样对比两个数据源 |

### 4.3 Tushare 积分问题

| 接口 | 所需积分 | 用途 |
|------|---------|------|
| `fina_indicator`（财务指标） | 200 | 基本面/成长性/安全性 |
| `daily_basic`（每日指标） | 200 | PE/PB |
| `daily`（日线行情） | 200 | 趋势/波动率 |
| `index_weight`（指数成分股权重） | 2000 | **历史成分股快照** |
| `disclosure_date`（财报发布日期） | 2000 | **前视偏差消除** |

基础接口 200 积分即可（注册即送），但历史成分股和财报发布日期需要 **2000 积分**。

**获取 2000 积分的途径**：
- 注册 + 完善个人信息：约 100-200 分
- 绑定手机/邮箱/GitHub：约 50-100 分
- 社区贡献（发帖、回答问题）：可累积
- 推荐新用户：50 分/人
- 付费购买：最直接（具体价格需查询 Tushare 官网）

## 五、推荐方案

### 5.1 整体策略：AKShare 为主 + Tushare 补关键缺口

```
日常数据采集（每日运行）          回测专用数据（一次性获取）
┌─────────────────────┐     ┌─────────────────────────┐
│ AKShare（免费，无门槛）│     │ Tushare（需 2000 积分）    │
│                       │     │                           │
│ • 日线行情            │     │ • 沪深 300 历史成分股快照   │
│ • 财务报表            │     │ • 财报实际发布日期         │
│ • PE/PB               │     │ • （作为交叉验证备选源）   │
│ • 行业分类            │     │                           │
│ • 资金流向            │     │                           │
│ • 当前成分股列表      │     │                           │
│ • 无风险利率          │     │                           │
└─────────────────────┘     └─────────────────────────┘
```

**逻辑**：
- AKShare 完全免费，覆盖了日常运行 95% 的数据需求，作为主力数据源
- Tushare 只用于补两个 AKShare 缺失且回测必须的字段：历史成分股 + 财报发布日期
- Tushare 回测数据是一次性获取后本地缓存，不需要频繁调用

### 5.1.1 Z-Score 标准化范围：全市场同行业

**问题**：沪深 300 在部分申万一级行业中只有 2-3 只股票（如纺织服饰、轻工制造、农林牧渔等），Z-Score 样本量不足，排名无意义。

**结论**：Z-Score 标准化使用**全市场同行业**数据（约 5000 只 A 股），推荐范围仍限沪深 300。

```
数据拉取范围：全市场 ~5000 只 A 股
Z-Score 计算范围：全行业（每个申万一级行业所有股票参与排名）
战力评分输出范围：仅沪深 300
推荐 / 置换范围：仅沪深 300
```

**类比**：高考全省排名，但只从前 300 名里录取。

**数据量评估**：

| 数据 | 规模 | SQLite 承受力 |
|------|------|-------------|
| 最新一期财务指标（5000 股 × 15 字段） | ~7.5 万条 | 轻松 |
| 当日 PE/PB（5000 股 × 2 字段） | ~1 万条 | 轻松 |
| 1 年日线行情（5000 股 × 250 天） | ~125 万行 | 没问题 |
| 5 年日线行情（回测用） | ~625 万行 | SQLite 百万级完全可承受 |

**时间成本**：

| 操作 | 耗时 | 频率 |
|------|------|------|
| 首次全量拉取（5 年历史） | 1-2 小时 | 一次性 |
| 每日增量更新（全市场当日数据） | 5-10 分钟 | 每日 |
| 季度财报更新 | 10-20 分钟 | 每季度 |

**注意**：AKShare 拉取 5000 只股票需控制请求频率（每次间隔 0.3-0.5 秒），并实现重试逻辑，避免触发隐式限流。

### 5.2 分阶段执行计划

| 阶段 | 数据源 | 任务 | 积分要求 |
|------|--------|------|---------|
| **V1 数据验证** | AKShare only | 跑通沪深 300 所有字段，确认可用性 | 0 |
| **V2 战力计算** | AKShare only | 计算当前战力排名 | 0 |

### 5.3 DataProvider 接口设计

无论使用哪个数据源，业务代码不应直接依赖具体接口。通过抽象层隔离：

```python
class DataProvider(ABC):
    """数据源抽象接口"""
    
    @abstractmethod
    def get_daily_quotes(self, stock_code: str, start_date: str, end_date: str) -> pd.DataFrame:
        """日线行情：开盘/收盘/最高/最低/成交量/成交额"""
        
    @abstractmethod
    def get_financial_indicator(self, stock_code: str, period: str) -> dict:
        """财务指标：ROE/净利润率/毛利率/资产负债率/流动比率等"""
        
    @abstractmethod
    def get_pe_pb(self, stock_code: str, date: str) -> dict:
        """估值指标：PE(TTM)/PB"""
        
    @abstractmethod
    def get_money_flow(self, stock_code: str, date: str) -> dict:
        """资金流向：主力净流入"""
        
    @abstractmethod
    def get_index_constituents(self, index_code: str, date: str = None) -> list:
        """指数成分股（date=None 返回当前，否则返回历史）"""
        
    @abstractmethod
    def get_industry_classification(self, stock_code: str) -> str:
        """申万一级行业分类"""
        
    @abstractmethod
    def get_risk_free_rate(self, date: str) -> float:
        """无风险利率（10年期国债收益率）"""
```

日常运行时注入 `AKShareProvider`，回测时注入 `TushareProvider`（或混合使用）。

### A 股 TTM 通用计算公式

**关键背景**：A 股季报为年内累计值（Q2 报告含 1-6 月累计，Q3 含 1-9 月累计），不能直接将 4 个季报值相加。

```python
def calc_ttm(latest_annual, latest_quarterly_cum, prior_year_same_quarterly_cum):
    """
    A 股标准 TTM 公式
    
    参数：
      latest_annual: 最新年报值（完整 12 个月）
      latest_quarterly_cum: 最新季报累计值（如 2025Q3 = 1-9月累计）
      prior_year_same_quarterly_cum: 去年同期季报累计值（如 2024Q3 = 1-9月累计）
    
    示例（最新数据为 2025Q3）：
      TTM = 2024年报(1-12月) + 2025Q3(1-9月) - 2024Q3(1-9月)
          = 2024(10-12月) + 2025(1-9月) = 最近 12 个月
    
    特殊情况：
      如果最新数据就是年报（Q4），TTM = 年报值本身
    """
    if latest_quarterly_cum is None:  # 最新数据就是年报
        return latest_annual
    return latest_annual + latest_quarterly_cum - prior_year_same_quarterly_cum
```

**缓存需求**：`financial_indicators` 需存储**最新年报 + 最近 2 年各季度累计值**，以支撑 TTM 计算和 TTM 同比。

### 自由现金流计算公式（TTM）

```python
OCF_TTM = calc_ttm(年报_经营现金流, 最新季报累计_经营现金流, 去年同期季报累计_经营现金流)
CapEx_TTM = calc_ttm(年报_资本支出, 最新季报累计_资本支出, 去年同期季报累计_资本支出)
自由现金流_TTM = OCF_TTM - CapEx_TTM
```

### 利息保障倍数计算公式（TTM）

```python
利润总额_TTM = calc_ttm(年报_利润总额, 最新季报累计_利润总额, 去年同期季报累计_利润总额)
财务费用_TTM = calc_ttm(年报_财务费用, 最新季报累计_财务费用, 去年同期季报累计_财务费用)

if 行业 in [银行, 非银金融]:
    利息保障倍数 = None  # 金融行业跳过，安全性给基准分 50
elif 财务费用_TTM <= 0:
    利息保障倍数 = 999  # 无负债或利息收入 > 支出，极其安全
else:
    利息保障倍数 = (利润总额_TTM + 财务费用_TTM) / 财务费用_TTM
```
