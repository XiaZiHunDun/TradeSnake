# 股票分析软件本地数据存储与时效清理方案
## 一、整体设计原则
1. **分级存储**：高频实时数据轻量化，历史数据压缩归档，无用数据定时销毁
2. **按时效性分层**：越久远的数据保留粒度越粗、占用空间越小
3. **可配置化**：保留周期、清理阈值支持用户/后台自定义，不强制硬编码
4. **安全兜底**：清理前做校验，避免误删用户标记/自选/策略相关数据
5. **性能优先**：清理任务放在低峰期执行，不影响行情、交易、盘口响应

---

## 二、数据分类与存储策略
### 1. 实时行情类数据（占用最大、增长最快）
- **盘口/逐笔/逐笔成交**
  - 存储：内存缓存 + 本地轻量数据库（SQLite/LevelDB）
  - 时效：**仅保留当日**，收盘后自动清理
  - 例外：用户手动标记“重点盯盘”标的可延长至3日

- **分时图数据（1分钟/5分钟K线）**
  - 存储：结构化压缩存储，按股票代码分表/分文件
  - 时效：
    - 近 **7天**：完整保留原始精度
    - 7天～3个月：保留1/2采样（如只存整点/关键点位）
    - 超过3个月：**自动归档压缩**，仅保留关键指标（开盘/收盘/最高/最低/成交量）

### 2. K线历史数据（核心分析数据）
按周期分级保留：
- **1/3/5/15分钟K线**
  - 近 **30天**：完整保留
  - 1～3个月：降采样合并
  - 超过3个月：仅保留日K及以上级别

- **日K/周K/月K**
  - 日K：默认保留 **最近3年**
  - 周K/月K：**永久保留**（体积极小，不影响增长）
  - 可选：支持用户开启“全量历史”模式，额外占用需提示

### 3. 基础信息类数据（低频更新）
- 股票基本资料、行业概念、财务摘要、分红送配
- 存储：结构化表，增量更新
- 时效：**永久保留**，仅做更新覆盖，不删除
- 更新频率：每日盘后一次即可

### 4. 用户行为与策略数据（严禁随意清理）
- 自选股、持仓、交易记录、笔记、画线、策略回测结果
- 存储：独立库/表，与行情数据隔离
- 时效：**用户手动删除前永久保留**
- 机制：行情清理绝不触碰此区域，防止误删重要资产记录

### 5. 日志与缓存类垃圾数据
- 接口请求日志、错误日志、图片缓存、临时计算中间结果
- 时效：
  - 日志：保留 **7天**
  - 临时缓存：**每次启动自动清理**
  - 错误日志：超过30天自动删除

---

## 三、自动清理与增长控制机制
### 1. 触发方式
- **定时触发**：每日收盘后 15:30～16:00 自动执行
- **阈值触发**：
  - 本地数据目录大小超过设定值（如2GB/5GB/10GB可配置）
  - 单股票数据文件超过阈值（如100MB）
- **手动触发**：设置页提供“一键清理过期数据”按钮

### 2. 清理顺序（安全优先级）
1. 临时缓存、日志 → 2. 逐笔/盘口 → 3. 分时高频K线 → 4. 历史低频K线
**绝不先清理用户数据与基础资料**

### 3. 空间回收机制
- SQLite/文件型存储：清理后执行 **VACUUM** 压缩碎片
- 支持“仅清理指数/ETF/冷门股”，保留用户持仓与自选股数据
- 提供“轻度/中度/深度清理”三档模式：
  - 轻度：只清当日临时数据
  - 中度：清7天以上高频数据
  - 深度：清3个月以上明细数据

---

## 四、用户可配置项（建议界面实现）
- 本地数据最大占用空间上限
- 日K保留年限（1年/3年/5年/全部）
- 分时/分钟K保留天数
- 是否自动清理冷门股票数据
- 清理提醒开关（清理前弹窗确认）

---

## 五、异常与兜底机制
- 清理过程中断电/崩溃：下次启动自动校验，不产生损坏文件
- 磁盘空间不足：优先清理最旧、最非核心数据，强制释放空间
- 重要数据清理前：生成轻量备份（保留索引，不保留全量）
- 支持用户导出历史K线数据包，再执行清理，避免不可逆损失

---

## 六、极简落地版执行流程（可直接开发）
1. 启动软件 → 检查数据目录大小
2. 超过阈值 → 进入清理流程
3. 删除当日逐笔/盘口/临时缓存
4. 对7天以上分钟K线降采样/压缩
5. 对3个月以上高频K线只保留OHLC
6. 对3年以前日K提示是否归档
7. 执行数据库压缩，完成回收
