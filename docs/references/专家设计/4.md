# 股票分析软件本地数据存储方案

## 一、数据分类与时效性分析

股票数据根据时效性要求可分为以下几类：

| 数据类型 | 更新频率 | 建议保留时间 | 说明 |
|---------|---------|------------|------|
| 实时行情 | 分钟级/秒级 | 1-3天 | 用于短期技术分析 |
| K线数据（日/周/月） | 日更 | 长期保留 | 历史趋势分析必需 |
| 分时数据 | 分钟级 | 5-10天 | 用于日内分析 |
| 财务数据 | 季度 | 3-5年 | 基本面分析 |
| 公告/新闻 | 实时 | 30天 | 舆情分析 |
| 用户自选股 | 手动 | 永久 | 用户配置数据 |
| 分析结果缓存 | 按需 | 7-14天 | 避免重复计算 |

## 二、存储架构设计

### 1. 分层存储策略

采用「热温冷」分层架构：

```
热数据层（SSD/内存）
├── 实时行情（最近1-3天）
├── 当前交易时段数据
└── 频繁访问的技术指标缓存

温数据层（机械硬盘）
├── 近3个月日K数据
├── 近期分时数据（5-10天）
├── 财务数据（最近2年）
└── 用户分析历史

冷数据层（归档存储）
├── 历史K线数据（3年以上）
├── 归档财务报告
└── 历史公告存档
```

### 2. 数据库选型建议

```
├── SQLite：轻量级，适合本地个人用户，存储量 < 100GB
├── PostgreSQL：支持更复杂查询，适合专业用户
└── 时序数据库（InfluxDB）：专为时间序列数据优化，适合高频数据
```

## 四、数据清理策略

### 1. 自动清理规则

| 数据类型 | 清理条件 | 清理时机 |
|---------|---------|---------|
| 实时行情 | 超过3天 | 每日收盘后 |
| 分时数据 | 超过10天 | 每周一凌晨 |
| 缓存数据 | 超过7天未访问 | 每次启动时检查 |
| 临时文件 | 超过24小时 | 每次启动时 |

### 2. 清理实现示例

```python
import os
import time
from datetime import datetime, timedelta

class DataCleanupManager:
    def __init__(self, data_root: str):
        self.data_root = data_root
        self.retention_rules = {
            'quotes': 3,        # 天
            'ticks': 10,        # 天
            'cache': 7,         # 天
            'temp': 1,          # 天
        }
    
    def cleanup_by_retention(self):
        """基于保留期限清理"""
        for data_type, days in self.retention_rules.items():
            cutoff = time.time() - (days * 86400)
            data_path = os.path.join(self.data_root, data_type)
            
            for root, dirs, files in os.walk(data_path):
                for file in files:
                    file_path = os.path.join(root, file)
                    if os.path.getmtime(file_path) < cutoff:
                        os.remove(file_path)
                        print(f"已清理: {file_path}")
    
    def cleanup_by_size(self, max_size_gb: float = 50):
        """基于存储空间限制清理"""
        current_size = self.get_folder_size(self.data_root) / (1024**3)
        
        if current_size > max_size_gb:
            # 从最老的数据开始删除，直到达到目标大小
            oldest_files = self.get_all_files_sorted_by_date(self.data_root)
            target_size = max_size_gb * 0.8  # 保留20%余量
            
            for file_path in oldest_files:
                if current_size <= target_size:
                    break
                current_size -= os.path.getsize(file_path) / (1024**3)
                os.remove(file_path)
    
    def get_folder_size(self, path: str) -> int:
        total = 0
        for root, dirs, files in os.walk(path):
            for f in files:
                total += os.path.getsize(os.path.join(root, f))
        return total
```

## 五、数据同步策略

### 1. 更新策略

```

class StorageMonitor:
    def check_storage_health(self):
        """存储健康检查"""
        warnings = []
        
        # 检查存储使用率
        usage = self.get_storage_usage()
        if usage > 80%:
            warnings.append("存储使用率超过80%")
        if usage > 95%:
            warnings.append("严重：存储即将满，请立即清理")
        
        # 检查异常数据
        old_data_count = self.count_old_unupdated_data()
        if old_data_count > 1000:
            warnings.append(f"存在{old_data_count}条长期未更新数据")
        
        return warnings
```
