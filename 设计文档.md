# 股市贪吃蛇 (TradeSnake) - 初始设计文档

## 一、项目愿景

将股票市场重新诠释为一场**贪吃蛇游戏**。

传统的炒股思维是"低买高卖赚差价"，而本项目的核心理念是：**股市不是一个"赚钱"的地方，而是一个"价值置换与进化"的战场**。

> 类比：你的初始资金不是"钱"，而是一股**初始战力**。股市中每一只股票都有其"战力值"。你的目标不是持有更多金钱，而是通过不断**置换**——将低战力股票置换为高战力股票——来**持续提升自己的总战力**，最终成为战力最强的贪吃蛇。

---

## 二、核心概念映射

| 贪吃蛇游戏 | 股市实践 | 说明 |
|-----------|---------|------|
| 贪吃蛇 | 投资者自身 | 进入市场的交易者 |
| 蛇身长度 | 总战力值 | 衡量投资者实力的核心指标 |
| 食物 | 高价值股票 | 战力提升的来源 |
| 蛇头方向 | 市场趋势 | 指引贪吃蛇前进的方向 |
| 撞墙/撞自己 | 亏损/被套 | 游戏失败的代价 |
| 地图 | 股票市场全貌 | 所有可交易股票的集合 |
| 蛇生长 | 战力提升 | 通过成功置换实现 |
| 蛇缩短 | 战力下降 | 持有贬值股票 |

## 三、核心价值体系

### 3.1 战力值 (Combat Power / CP)

**战力值**是本项目的核心价值计量单位。

- 每只股票都有其**股票战力值 (Stock CP)**
- 投资者的**个人战力值 = 持有股票的战力值之和**
- 战力值不再以金钱衡量，而是基于股票的**内在价值因子**计算

### 3.2 战力值计算逻辑（初步设想）

```
股票战力值 = 基础因子 × 成长因子 × 稀缺因子

- 基础因子：股票的基本面价值（PE、ROE、营收增长率等）
- 成长因子：股票的成长性指标（净利润增速、市场份额变化）
- 稀缺因子：股票的稀缺程度（行业壁垒、市场地位、不可替代性）
```

### 3.3 置换机制

**置换**是本项目的核心操作：

```
低战力股票 + 手续费/时间成本 → 高战力股票
```

- **同板块置换**：同一行业内，将战力低的股票换成战力高的
- **跨板块置换**：将整个板块视作一个战力单位，进行板块间置换
- **逆势置换**：在市场下跌时，将账面战力暂时降低，换入未来潜力更高的股票

## 四、贪吃蛇的价值判定规则

### 4.1 最具战力股票榜单

- 系统实时计算并展示**战力榜 TOP 100**
- 战力榜每分钟/每日更新（可配置）
- 战力榜揭示了市场中最"值得置换"的目标

### 4.2 战力趋势

- 每只股票有**战力趋势图**：上升↗、平稳→、下降↘
- 战力上升的股票是"食物"
- 战力下降的股票是"毒药"

### 4.3 战力对比

- 个股详情页展示该股票与同行业其他股票的战力对比
- 类似于贪吃蛇游戏中"前方有更大的蛇/食物"的提示

## 九、战力公式详细设计

### 9.1 当前公式：v18（综合版）

```
总战力 = (成长分×40% + 价值分×40% + 动量分×20%) × 风险调整因子
```

| 因子 | 权重 | 数据来源 | 计算方式 |
|-----|------|---------|---------|
| **成长分** | 40% | 财报 | 净利润增速(0-300%)×0.6 + 营收增速(-50-100%)×0.4 |
| **价值分** | 40% | 实时行情 | ROE（负值当0，ROE>25%截断） |
| **动量分** | 20% | 行情数据 | 当日涨跌幅（限制在-10到10之间） |
| **风险调整** | - | 综合 | 低风险股票加权更高，高风险股票加权降低 |

**注意**：
- v18 公式简化了质量分，整合到成长和价值中
- 标准化方式：Min-Max归一化到0-100分
- 风险因子：`1 - (风险分数/100) × 0.10`

### 9.2 v18 公式各因子详解

#### 成长分计算
```python
net_g = max(0, min(300, self.net_profit_growth))  # 净利润增长：限制0-300%
rev_g = max(-50, min(100, self.revenue_growth))    # 营收增长：限制-50到100%
growth_score = net_g * 0.6 + rev_g * 0.4
```

#### 价值分计算
```python
base_roe = min(max(0, self.roe), 25)  # ROE限制在0-25
pe_score = ...  # PE健康度评分
peg_bonus = ...  # PEG估值评分
pb_score = ...  # PB市净率评分
value_score = max(0, base_roe + pe_score + peg_bonus + pb_score * 0.3)
```

#### 质量分计算
```python
cf_score = ...  # 现金流/ROE比例评分
gm_score = ...  # 毛利率评分
debt_score = ...  # 资产负债率评分
quality_score = max(0, cf_score + gm_score + debt_score)
```

#### 动量分计算
```python
momentum_score = max(-10, min(10, self.change_pct))  # 当日涨跌幅，限制在-10到10
```

### 9.3 v15 交易成本建模（重要！）

**问题**：原公式没有考虑股票交易的真实成本。

**v15 解决方案**：将交易成本纳入换股决策。

#### 交易成本常量
```python
TRADE_COST = {
    'commission': 0.0003,       # 券商佣金：万分之三
    'stamp_tax': 0.0005,        # 印花税：万分之五（仅卖出时收取）
    'transfer_fee': 0.00001,     # 过户费：十万分之一（沪市双向，深市免）
    'min_commission': 5.0,       # 最低佣金：5元/笔
}

# 完整换股一次（买+卖）的总成本比率
SELL_COST_RATE = 0.081%  # 卖出：佣金+印花税+过户费
BUY_COST_RATE = 0.031%   # 买入：佣金+过户费
TOTAL_TRADE_COST_RATE = 0.112%  # 总计约112元/10万元
```

#### 现金战力（CashCP）
```python
class CashCP:
    """现金战力 = 持有现金的机会成本"""
    RISK_FREE_RATE = 0.02  # 年化无风险利率 2%

    @classmethod
    def get_opportunity_cost(cls, cash: float, days: int) -> float:
        """
        公式：cash × (年化2% / 365) × 天数
        示例：10万现金持有30天 = 100000 × 0.000055 × 30 = 164.38元
        """
        daily_rate = cls.RISK_FREE_RATE / 365
        return cash * daily_rate * days
```

#### 换股决策（TradeDecision）
```python
class TradeDecision:
    """
    核心公式：
    换股净收益 = (B战力 - A战力) × 本金 × (持有天数/365) - 交易成本

    换股建议分级：
    - 🟢 强烈建议换：净收益 > 交易成本的120%
    - 🟡 可以考虑换：净收益 > 0
    - ⚫ 持有不动 🎯：净收益 <= 0
    - 🔴 别换！：净收益 < -交易成本的50%
    """
```

#### 战力差阈值
| 资金量 | 持有30天阈值 | 说明 |
|--------|-------------|------|
| 1万 | > 11分 | 需要很大战力差才能回本 |
| 5万 | > 2.2分 | 最低消费影响显著 |
| 10万 | > 1.1分 | 相对合理 |
| 50万 | > 0.2分 | 大资金换股几乎无门槛 |

**核心结论**：小资金做短线几乎不可能赚钱（被交易成本吃掉）！

### 9.3 个人战力计算

```
个人总战力 = Σ(持有股票数量 × 该股票战力值)
```

也就是说：你持有100股某股票，该股票战力值80，那这只股票对你的战力贡献就是 100 × 80 = 8000。

**战力变化** = 今日战力 - 昨日战力（类似于贪吃蛇的长度变化）
