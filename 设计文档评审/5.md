## 一、概念设计评审

**优点：**
- **叙事统一性强**：将“低买高卖”重新定义为“战力置换”，与贪吃蛇的“吃食物变长”形成完整映射，逻辑自洽。
- **战力值作为核心锚点**：避免了传统金融指标的割裂感，让用户只关注一个核心数字，符合游戏化设计原则。
- **置换机制**：“低战力→高战力”的表述比“卖出/买入”更符合贪吃蛇的“吞食”逻辑，有助于建立心理账户隔离。

**建议考虑：**
- **“战力值”与“资金”的关系需进一步明确**：当前设计里，买入股票需要消耗资金，但资金在战力体系中如何体现？如果资金本身不是战力，那么用户可能需要“先用资金换低战力股，再置换为高战力股”，中间环节可能会产生认知断层。
  - 建议：明确**资金 = 初始战力基数**，每次买入是将资金“转化”为对应股票的战力值，卖出则是将战力值“还原”为资金（但可能有损耗）。

## 二、数据与计算评审

**优点：**
- **v1.0公式简洁可落地**：用三个因子（成长、价值、趋势）加权，标准化为0-100分，实现成本低。
- **百分位排名标准化**：避免了不同量纲因子的直接相加问题。
- **个人战力 = 持股数量 × 战力值**：逻辑清晰，且符合“蛇身长度”的直观感受。

**潜在问题：**
1. **PE负值或极高值的处理**  
   - 当前策略是“取低值”，但未给出具体数值。如果直接按百分位排名，负PE会排在末尾（合理），但极高PE（如1000倍）也可能排名靠后（也合理）。建议明确边界处理逻辑，避免因个别股票数据异常导致战力值突变。

2. **新股处理**  
   - 上市不足1年的股票用“营收增速”替代，但营收增速可能极高（如刚上市的公司），导致战力值虚高。建议对新股增加**惩罚系数**或单独标记。

3. **数据更新频率**  
   - v1为日更，但财报数据是季更，趋势分是日更，建议在战力趋势图中区分“财报更新日”与“日常估值波动”，否则用户可能会误解战力变化的稳定性。

## 三、技术与实现评审

**优点：**
- **技术栈选择务实**：Python + FastAPI + React + SQLite，适合单人开发，快速迭代。
- **手动录入持仓**：避免了券商对接的合规与技术成本，符合“私人工具”定位。
- **v1功能边界清晰**：未过度设计，MVP目标明确。

**建议调整：**
1. **数据源选择**  
   - 文档中写“东方财富/同花顺，本地即可获取”。实际上这两家并没有公开的、可直接调用的免费API。目前A股数据常用的免费/低成本方案包括：
     - **Tushare**（需积分，A股数据较全）
     - **AkShare**（开源Python库，直接抓取，但稳定性依赖第三方）
     - **baostock**（免费，无积分限制，数据质量尚可）
   - 建议：v1直接用 **AkShare + 本地缓存**，先保证能跑通，后续可切换更稳定数据源。

2. **SQLite并发与扩展性**  
   - 单人使用完全够用，但需注意：如果后续想加定时任务（如每日自动拉取数据），需处理好数据库锁的问题。建议用 `sqlite3` + `WAL` 模式。

3. **前端游戏化动画性能**  
   - ECharts 擅长图表，但蛇身卡片、吞食动画等需要自己用 CSS/Canvas 实现。React 的频繁重绘可能影响动画流畅度，建议蛇身交互部分用 Canvas 或 CSS Transform 独立处理。

## 四、用户体验评审

**优点：**
- **游戏化UI设计生动**：能量条、段位、蛇身卡片，与核心概念高度一致。
- **交互反馈明确**：买入/卖出有动画，战力变化有飘字，符合游戏心理。

**建议优化：**
1. **“段位”与“战力值”的对应关系**  
   - 目前段位是固定区间，但实际市场中，A股全市场持仓500万和5万的人战力差距巨大。如果用户初期持仓少，可能长期困在“青铜”，容易产生挫败感。
   - 建议：段位可以改成**相对段位**（例如：超过X%的玩家），或者增加**每周战力进步榜**，让用户更关注自身成长而非绝对排名。

2. **“蛇身”与“持仓”的映射**  
   - 设计中蛇身是横向卡片列表，但贪吃蛇的蛇身是有顺序的（头→身→尾）。当前设计只是展示持仓，没有体现“顺序”。可以考虑**按战力从低到高排列**，尾部是低战力股（待置换），头部是高战力股（最终目标），与贪吃蛇逻辑对齐。

3. **“食物”与“毒药”的视觉区分**  
   - 设计中有战力榜，但用户需要主动去“找食物”。可以在个人面板中增加一个**推荐猎物**区域，展示当前战力榜前列且与用户持仓同行业的股票，降低决策成本。

## 五、风险与后续迭代建议

| 风险 | 影响 | 应对建议 |
|------|------|---------|
| **战力公式被滥用** | 用户可能只买“高分股”，忽略市场周期和流动性 | v1阶段不涉及交易建议，仅展示，问题可控；后续若加入置换建议，需加入风险提示 |
| **数据源不稳定** | 可能导致战力计算失败或偏差 | 增加数据缓存机制，财报数据可缓存一周，行情数据每日拉取失败时使用昨日值并标记 |
| **手动录入体验差** | 用户可能懒得维护持仓 | v1阶段先验证战力计算本身是否有趣，后续可支持导入券商交割单（CSV） |
